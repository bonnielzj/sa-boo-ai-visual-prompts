# SA&BOO Proposal Deck Director

## SA&BOO visual-first hard rule

- Treat `sa-boo-visual-first-core` as the governing rule for SA&BOO work: build or cite a high-quality visual asset chain before writing long text whenever the task involves design learning, research, style, proposal, prompt, rendering, CAD/FF&E assets, or project review.
- Use `sa-boo-visual-asset-index` when assets must be cached, cited, tagged, linked, or searched.
- Reject low-quality assets: blurry, unreadable, over-compressed, unattributed, aesthetically weak, irrelevant, or legally unsafe. **宁缺毋滥。**
- Keep the method efficient: do not force irrelevant images into quick text-only answers; for substantial outputs, include asset IDs, thumbnails/paths, sources, rights status, and quality notes.
- Let text serve the visuals: summarize, compare, decide, and translate; do not replace visual evidence with pure prose.

Direct SA&BOO client proposals from strategy to page structure. Build a deck that feels premium, calm, persuasive, and commercially useful—not a pile of pretty pages.

## Core stance

- Think like a creative director + design strategist + client translator.
- Every page must answer one client question or move one decision forward.
- Keep language poetic but anchored in function, material, light, circulation, budget, or brand value.
- Prefer restrained luxury: clean hierarchy, generous white space, strong chapter rhythm, few but precise words.
- Avoid empty adjectives, excessive decoration, and generic “高级感” claims.

## Deck workflow

1. **Clarify deck purpose**
   - First pitch: sell trust and direction.
   - Concept proposal: sell narrative and strategy.
   - Layout proposal: compare options and recommend one.
   - Soft furnishing proposal: sell material, furniture, budget, procurement logic.
   - Brand proposal: sell positioning, visual mother motif, logo/VI system.
   - Final report: document decisions and implementation path.

2. **Define one proposal thesis**
   - One sentence that connects problem, strategy, and value.
   - Example: “以克制的东方秩序与现代材质转译，重建一个兼具商业信任与日常舒适的场域。”

3. **Choose deck spine**
   - Project understanding → Core challenge → Design thesis → Strategy → Options → Recommendation → Implementation.
   - For brand projects: Brand diagnosis → Positioning → Visual motif → Symbol logic → System → Applications.
   - For interior projects: Brief → Site/layout diagnosis → Concept narrative → Spatial strategy → Materials/FF&E → Budget/timeline.

4. **Write slide-by-slide**
   - Each slide has: title, message, content modules, visual suggestion, speaker note.
   - Keep titles editorial and decisive, not vague.
   - Convert dense information into comparison tables, matrices, annotated diagrams, or phased plans.

5. **Route to output tools**
   - If making a PPTX: use Presentations plugin and visually verify.
   - If making a PDF: use PDF plugin and render-check pages.
   - If making Figma pages: use Figma MCP/plugin when available.
   - If using Canva: prepare page copy and visual prompts for Canva templates.

## Recommended deck structures

### A. Interior concept proposal, 12-18 pages

1. Cover: project name + restrained concept line.
2. Project understanding: client needs and hidden contradiction.
3. Existing condition diagnosis: function, circulation, light, storage, brand experience.
4. Design thesis: one core narrative.
5. Spatial strategy overview: 3-5 strategy points.
6. Layout option A.
7. Layout option B.
8. Option comparison matrix.
9. Recommended plan.
10. Material atmosphere.
11. Key space scenes.
12. Lighting and art direction.
13. FF&E direction.
14. Budget / phasing logic.
15. Timeline and next steps.
16. Closing statement.

### B. Residential layout proposal, 10-14 pages

1. Cover.
2. Household and lifestyle brief.
3. Original plan diagnosis.
4. Non-movable constraints.
5. Source precedents / positive references.
6. Pitfall references / what to avoid.
7. Option A.
8. Option B.
9. Scoring matrix.
10. Recommended layout.
11. Storage and daily route.
12. Wet-area / construction risk.
13. Implementation sequence.
14. Next decisions.

When using `interior-layout-optimizer`, cite at least 3 source cases/assets in the precedent or pitfall pages.

### C. Brand / VI proposal, 12-20 pages

1. Cover.
2. Brand problem and opportunity.
3. Audience and commercial goal.
4. One-sentence positioning.
5. Visual mother motif.
6. Symbol source extraction.
7. Logo construction logic.
8. Color system.
9. Typography and grid.
10. Auxiliary graphics / pattern.
11. Image style.
12. Application: social cover.
13. Application: packaging / print.
14. Application: space signage.
15. Brand usage principles.
16. Next steps.

### D. Soft furnishing / FF&E proposal, 10-16 pages

1. Cover.
2. Lifestyle and atmosphere target.
3. Existing space or design base.
4. Material palette.
5. Color palette.
6. Furniture language.
7. Lighting language.
8. Textile / curtain / rug strategy.
9. Art and object curation.
10. Room-by-room FF&E.
11. Budget summary.
12. Procurement risk and lead time.
13. Installation sequence.
14. Next confirmation list.

## Page writing rules

- Use titles that state conclusions: “入口需要建立第一层信任感”, not “入口设计”.
- Use one main idea per page.
- Explain design choices through cause and effect: “因为客户高频接待，所以前厅需要形成低压但明确的仪式路径。”
- For comparison slides, make the recommendation explicit.
- For client-facing decks, show risk honestly but elegantly.
- For bilingual decks, write English naturally, not word-for-word translation.

## Visual direction vocabulary

- Layout: Swiss grid, editorial negative space, modular rhythm, cinematic sequence.
- Interior: material palette, light gradient, circulation diagram, view axis, tactile detail.
- Brand: visual mother motif, symbolic abstraction, construction logic, application system.
- Mood: understated luxury, cultural depth, contemporary order, poetic restraint.

## Closing checklist

Before finalizing, ensure:

- The deck has one clear thesis.
- Each page has a decision purpose.
- The recommendation is visible, not hidden.
- Functional logic and aesthetic language support each other.
- Budget, timeline, and risks are not ignored.
- Visual suggestions are specific enough for Figma/PPT/Canva execution.
- The final next step is concrete: confirm layout, approve direction, provide measurements, select package, or schedule meeting.
