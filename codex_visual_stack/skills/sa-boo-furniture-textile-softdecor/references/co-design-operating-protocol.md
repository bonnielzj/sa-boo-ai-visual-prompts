# SA&BOO Co-Design Operating Protocol

Use this whenever Bonnie asks for maximum skill/resource mobilization, design collaboration, or says the goal is design rather than writing.

## Core promise

Work as a design partner. Do not merely describe. Use text to drive action and decisions.

```text
research → visual evidence → design options → CAD/SU/material/FF&E translation → QA → next artifact
```

## Resource hierarchy

Use as much as is relevant to improve quality:

1. **User/local assets** — project files, screenshots, CAD/SU, local warehouse, visual asset index.
2. **Official brand sources** — product pages, CAD/SKP/DWG/Revit/BIM/3D downloads, catalogues, materials, scene images.
3. **SA&BOO skills** — furniture/system, visual-first, CAD↔SU, SketchUp director, material/supplier DB, proposal, render, research.
4. **MCP/tools** — AutoCAD, cadq, browser/search, local scripts, image/document/spreadsheet tools when useful.
5. **Open web / GitHub / datasets** — only when license and relevance are clear; record source and rights.
6. **Standards and certifications** — textile, furniture, accessibility, emissions, sustainability.

## Buildability and precision are co-equal with beauty

Design must land in the real world. CAD and SketchUp precision, dimensions, scale, construction logic, layer/tag discipline, model hygiene, and issue-ready drawing expression are not afterthoughts. A beautiful idea that cannot be dimensioned, modeled, coordinated, specified, or built is not complete SA&BOO design.

For any project intended to be built or presented as feasible:

- verify units, scale, known dimensions, and clearances;
- connect furniture/system choices to CAD footprints, elevations, sections, module schedules, and tags;
- connect SketchUp models to clean components, correct tags, material slots, LOD/proxy strategy, and import hygiene;
- distinguish visual concept from construction truth;
- flag unverified dimensions, brand model limits, site-measure needs, and installation risks;
- use `sa-boo-cad-sketchup-bridge`, `sa-boo-autocad-construction-standard`, `autocad-drafting`, `cadq`, and SketchUp QA skills whenever precision affects quality.

## Co-design behaviors

- Create options, not just explanations.
- Compare and reject weak options openly.
- Pull decisions back to proportion, material, scale, construction, light, human ritual, and feasibility.
- Translate beauty into CAD/SU/material/FF&E consequences.
- Ask for missing visual/project evidence only when it blocks quality; otherwise make explicit assumptions and move.
- End with the next concrete artifact: image capture, model action, CAD action, source card, schedule row, prompt, or QA list.

## Anti-writing rule

Do not produce long standalone prose when one of these would be more useful:

- moodboard or visual chain;
- annotated reference list;
- product/system comparison table;
- material board;
- CAD/SU execution list;
- FF&E schedule;
- render prompt and camera list;
- model/drawing QA checklist.
