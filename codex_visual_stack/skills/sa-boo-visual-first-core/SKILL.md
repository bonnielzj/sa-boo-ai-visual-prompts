# SA&BOO Visual-First Core

This is the governing doctrine for SA&BOO skills: **视觉链表为主，文字为辅**. A SA&BOO output should be anchored by clear, beautiful, readable, traceable visual evidence whenever the task involves design judgment, learning, research, style, proposal, prompt, project review, rendering, CAD, materials, brand, or content.

## Hard rule

- Build or cite a **high-quality visual asset chain** before long-form explanation when visuals would improve judgment, learning, or delivery.
- Use text as interpretation, structure, decision logic, narrative, and QA. Do not let pure prose replace visual evidence.
- Reject low-quality assets: blurry, unreadable, over-compressed, watermarked beyond usefulness, unattributed, aesthetically weak, irrelevant, or impossible to reuse as study evidence.
- Prefer original or stable sources: official galleries, brand/project pages, supplier catalogs, high-resolution screenshots, exported project images, renders, diagrams, and verified local files.
- Keep the method efficient and elegant. Do not force irrelevant images into quick text-only answers; state the visual status and continue with best judgment.


## SA&BOO DNA

Carve this into every SA&BOO workflow:

- **Aesthetic intelligence** — pursue refined, contemporary, evolving beauty; never let yesterday's style define tomorrow's output.
- **Design rigor** — use proportion, grid, spatial logic, material logic, construction feasibility, source evidence, and QA rather than vague taste claims.
- **Visual-first evidence** — lead with high-quality, readable, traceable visual assets; let text interpret and sharpen the visual chain.
- **System over accident** — convert every project, reference, prompt, render, drawing, proposal, and social post into reusable standards, templates, assets, and review criteria.
- **Dream with discipline** — keep the work poetic and imaginative, but make every output beautiful, usable, commercial, and technically defensible.

Default SA&BOO ambition: **惊艳行业，但不依赖浮夸；创造梦境，但用严谨实现。**

## Asset-chain minimum

For substantial SA&BOO outputs, include or create these fields through `sa-boo-visual-asset-index` when assets are being collected or preserved:

```text
asset_id
thumbnail_path
local_original_path
source_url
original_image_url or source_file_path
capture_date
category / tags
rights_status
quality_notes
prev_id / next_id
related_ids
```

## Quality gate

Approve a visual asset only when it passes:

1. **Readability** — key forms, text, texture, scale, or construction logic are visible.
2. **Resolution** — image is sufficiently sharp for analysis, reference, or presentation.
3. **Attribution** — source, author/platform, or project origin is recorded when available.
4. **Aesthetic value** — composition, light, material, color, hierarchy, or craft teaches something useful.
5. **Relevance** — asset supports the exact design question, not generic decoration.
6. **Usage safety** — rights status is noted; third-party cached assets are private study references unless licensed.

If the asset fails, mark it as rejected and explain briefly. **宁缺毋滥。**

## Operating sequence

```text
brief / question
→ decide whether visual evidence is needed
→ collect, cite, cache, or create high-quality assets
→ link assets as a browsable chain
→ analyze visual logic and gaps
→ write concise supporting text
→ produce beautiful, structured output
→ record missing assets or next capture tasks
```

## How to combine with other skills

- Use `sa-boo-visual-asset-index` for caching, metadata, thumbnails, linked-list browsing, and retrieval.
- Use `sa-boo-design-research-radar` for current trend scans, then attach representative assets.
- Use prompt/style skills to generate controlled visual outputs, then keep the best results as assets with quality notes.
- Use proposal, brand, interior, render, CAD, material, and social skills only after deciding what visual evidence should lead the output.

## Output pattern

```text
视觉主线：assets / chain / source status
核心判断：what the visuals prove
文字辅助：brief, method, comparison, decisions
质量检查：clarity / attribution / rights / relevance
下一步：missing captures, better sources, or production tasks
```
