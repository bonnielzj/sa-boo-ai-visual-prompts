# Codex 工作规则

- 长任务、长文档、长日志、批量整理、图片/素材索引等内容，优先写入项目文件，不要把完整内容直接输出到聊天里。
- 聊天窗口只保留简短进度、关键判断、文件路径和下一步，避免触发上下文自动压缩失败。
- 需要保留过程证据时，创建清晰命名的 Markdown / JSON / CSV 文件，并在最终回复中引用路径。
- 如果任务会产生大量中间结果，先建立 `codex_outputs/` 或项目对应子目录，再分批写入。
- 新增或发现的长期可复用能力、skills、MCP 配置、工具使用经验、恢复摘要和关键工作规则，必须优先写入仓库内的持久化文件，再提醒用户推送到 GitHub。
- 不能只把重要上下文留在聊天里；如果当前仓库配置了 GitHub remote，并且用户明确允许推送，则在完成文件更新后执行 `git push`。
